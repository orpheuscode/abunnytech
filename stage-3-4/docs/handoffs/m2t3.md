## What Stage 4 emits and where to read it

### 1. Optimization Directives — `OptimizationDirectiveEnvelope`

**Where:** SQLite `directives` table (via `StateAdapter`), or in-memory via `Stage4Runner.run()`.

**How to consume (Stage 2 example):**

```python
from agents.stage4_analytics.state_adapter import StateAdapter
from datetime import datetime, timedelta

adapter = StateAdapter(db_path="./data/stage4_analytics.db")

# Pull directives targeting Stage 2 from the last 24h
directives = adapter.load_directives(
    target_stage="stage2",
    since=datetime.utcnow() - timedelta(days=1),
)

for d in directives:
    if d.directive_type == "hook_rewrite":
        # d.payload["hook_style"]     — preferred hook style
        # d.payload["avoid_styles"]   — styles to avoid
        # d.payload["example_hooks"]  — example text hints
        pass
    elif d.directive_type == "content_tier_rebalance":
        # d.payload["target_ratio"]   — {"hero": 50, "hub": 40, "hygiene": 10}
        pass
    elif d.directive_type == "audio_swap":
        # d.payload["preferred_audio_ids"]
        # d.payload["avoid_audio_ids"]
        pass
```

**Directive types and target stages:**

| directive_type | target_stage | Stage action |
|---|---|---|
| `hook_rewrite` | `stage2` | Change hook_style in VideoBlueprint generation |
| `content_tier_rebalance` | `stage2` | Adjust hero/hub/hygiene posting ratio |
| `audio_swap` | `stage1+stage2` | Discover better audio + use in blueprints |
| `schedule_shift` | `stage3` | Move posts to higher-performing slots |
| `product_focus` | `stage2` | Feature top-revenue products more prominently |

### 2. Redo Queue — `RedoQueueItem`

**Where:** SQLite `redo_queue` table (status=`queued`).

**How to consume:**

```python
adapter = StateAdapter(db_path="./data/stage4_analytics.db")
items = adapter.load_redo_queue(status="queued")

for item in items:
    # item.target_stage          — "stage2" | "stage3" | "stage1"
    # item.redo_reason           — why it was queued
    # item.suggested_mutations   — {hook_style, avoid_schedule_slot, ...}
    # item.source_video_blueprint_id — original blueprint to mutate from

    # After processing:
    adapter.mark_redo_done(item.item_id)
```

**Redo reasons and suggested_mutations keys:**

| redo_reason | target_stage | mutations |
|---|---|---|
| `hook_failed` | stage2 | `hook_style`, `hook_rewrite`, `target_3s_retention_pct` |
| `low_completion` | stage2 | `reduce_duration`, `add_midroll_hook`, `target_completion_pct` |
| `underperformed` | stage2 | `new_trend_angle`, `try_audio_swap`, `avoid_schedule_slot` |
| `wrong_schedule` | stage3 | `avoid_schedule_slot` |
| `audio_mismatch` | stage1 | `avoid_audio_id` |

## How Stage 4 consumes upstream data

Stage 4 reads `DistributionRecord` objects from Stage 3. The minimal required fields are:

```python
DistributionRecord(
    record_id="...",
    content_package_id="...",
    platform="tiktok",        # required
    post_id="post_abc123",    # required — if None, record is skipped
    status="posted",
    audio_id="...",           # optional context tag
    schedule_slot="tue_18:00" # optional context tag
)
```

Stage 2's `ContentPackage` provides `hook_style` and `content_tier` — Stage 3 should
propagate these into the distribution record or store them in a way Stage 4 can join on
`content_package_id`.

## Running the pipeline manually

```bash
# Fixture demo (no credentials needed)
python examples/stage4/run_fixture_demo.py

# Programmatic usage
import asyncio
from agents.stage4_analytics.runner import Stage4Runner

async def run():
    runner = Stage4Runner(dry_run=True)
    result = await runner.run(distribution_records)
    print(result.summary_daily)
    for d in result.directives:
        print(d.directive_type, d.priority, d.payload)

asyncio.run(run())
```
