# Handoff: m2t1 — Browser Runtime

All browser-facing stages import from `browser_runtime`.  This document is the integration contract.

---

## Core concepts

```
Stage code
   │
   ▼
PlatformAdapter          ← platform-specific knowledge (what to call, how to format)
   │
   ▼
BrowserProvider          ← execution backend (how to drive the browser)
   │
   ├── MockProvider       zero credentials, in-process, returns fixture data
   ├── BrowserUseProvider agent loop (browser-use + Playwright)
   ├── CodeAgentProvider  bulk extraction (smolagents CodeAgent)
   ├── SkillAPIProvider   pre-built macros via HTTP
   └── PlatformAPIProvider official REST/GraphQL APIs (TikTok, Instagram, Shopify)
```

**Golden rule:** stages never import from `providers/` or `adapters/` internals — only from `browser_runtime` top-level or `browser_runtime.types`.

---

## Quick start — mock (zero credentials)

```python
from browser_runtime import get_provider, get_adapter
from browser_runtime.types import PostContentRequest, Platform

provider = get_provider("mock", dry_run=True)
adapter = get_adapter("tiktok", provider)

result = await adapter.post_content(
    PostContentRequest(
        platform=Platform.TIKTOK,
        caption="AI content demo 🤖 #AI",
        dry_run=True,
    )
)
print(result.post_id)   # DRY-RUN-xxxxxxxx
print(result.success)   # True
```

## All request/response types

Import from `browser_runtime.types`.

| Type | Direction | Used by |
|------|-----------|---------|
| `PostContentRequest` / `PostContentResult` | Stage 3 → adapter | Distribute |
| `CommentReplyRequest` / `CommentReplyResult` | Stage 3 → adapter | Engage |
| `DMRequest` / `DMResult` | Stage 3 → adapter | Engage |
| `AnalyticsFetchRequest` / `AnalyticsData` | Stage 4 → adapter | Analyze |
| `TrendingFetchRequest` / `TrendingItem` | Stage 1 → adapter | Discover |
| `AgentTask` / `AgentResult` | any → provider | open-ended tasks |
| `ExtractionSchema` / `ExtractionResult` | Stage 1 → provider | bulk scrape |
| `SkillRequest` / `SkillResult` | any → provider | skill macros |
| `PlatformAPIRequest` / `PlatformAPIResponse` | adapter internals | official APIs |
| `SessionState` | session → persist | restore auth |
| `BrowserAction` / `ActionResult` | low-level | provider internals |

## Platform adapters

### TikTokAdapter

```python
from browser_runtime.adapters.tiktok import TikTokAdapter

adapter = TikTokAdapter(provider)

# Post a video
result = await adapter.post_content(PostContentRequest(...))

# Reply to a comment (persona-consistent)
result = await adapter.reply_to_comment(CommentReplyRequest(
    post_id="...", comment_id="...", reply_text="...", dry_run=True
))

# Trending audio for Stage 1
items = await adapter.fetch_trending(TrendingFetchRequest(
    platform=Platform.TIKTOK, niche_tags=["tech"], limit=10
))
```

**Live credentials:** `TIKTOK_ACCESS_TOKEN`, `TIKTOK_OPEN_ID`

### InstagramAdapter

```python
from browser_runtime.adapters.instagram import InstagramAdapter

adapter = InstagramAdapter(provider)
result = await adapter.post_content(PostContentRequest(
    platform=Platform.INSTAGRAM,
    media_url="https://...",  # must be publicly accessible
    caption="...",
))
```

**Live credentials:** `INSTAGRAM_ACCESS_TOKEN`, `INSTAGRAM_ACCOUNT_ID`

### AnalyticsAdapter

```python
from browser_runtime.adapters.analytics import AnalyticsAdapter

adapter = AnalyticsAdapter(provider)

# Single platform
data = await adapter.fetch_analytics(AnalyticsFetchRequest(
    platform=Platform.TIKTOK, post_id="..."
))

# Cross-platform (dashboard view)
summary = await adapter.cross_platform_summary({
    Platform.TIKTOK: "tt_post_id",
    Platform.INSTAGRAM: "ig_post_id",
})
```

### ShopifyAdapter (Stage 5 — feature-flagged)

```python
# MUST check feature flag before constructing
if feature_flags.is_enabled("stage5_monetize"):
    from browser_runtime.adapters.shopify import ShopifyAdapter
    adapter = ShopifyAdapter(provider)
    products = await adapter.list_products()
    await adapter.create_discount("LAUNCH20", "LAUNCH20", percentage=20.0)
```

**Live credentials:** `SHOPIFY_STORE_DOMAIN`, `SHOPIFY_ADMIN_TOKEN`

## Session management

```python
from browser_runtime.session import SessionManager

mgr = SessionManager()
session = mgr.create(platform="tiktok", dry_run=True)

tab = await session.create_tab("https://tiktok.com")
await tab.navigate("https://tiktok.com/upload")
content = await tab.get_content()

state = await session.save_state()   # snapshot cookies/storage
await session.restore_state(state)   # restore later

await mgr.close_all()                # clean teardown
```

## Kill switch

Flip in `.env` to stop all operations immediately:

```env
BROWSER_GLOBAL_KILL_SWITCH__ENABLED=true
BROWSER_GLOBAL_KILL_SWITCH__REASON="Demo paused — judge walkthrough"
```

Per-platform:

```env
BROWSER_TIKTOK__KILL_SWITCH__ENABLED=true
BROWSER_TIKTOK__KILL_SWITCH__REASON="TikTok rate limited"
```

All adapter methods raise `KillSwitchTriggered` immediately.

## Configuration reference

All settings via environment variables (prefix: `BROWSER_`):

| Env var | Default | Notes |
|---------|---------|-------|
| `BROWSER_DRY_RUN` | `true` | Set to `false` for live posting |
| `BROWSER_PROVIDER` | `mock` | `mock` \| `browser_use` \| `code_agent` \| `skill_api` \| `platform_api` |
| `BROWSER_AUDIT_LOG_PATH` | `./logs/browser_runtime_audit.jsonl` | JSONL audit log |
| `BROWSER_GLOBAL_KILL_SWITCH__ENABLED` | `false` | Emergency stop |
| `BROWSER_TIKTOK__RATE_LIMIT__REQUESTS_PER_MINUTE` | `10` | |
| `BROWSER_INSTAGRAM__RATE_LIMIT__REQUESTS_PER_MINUTE` | `10` | |

Nested settings use `__` as delimiter.

## AI disclosure enforcement

`PostContentRequest.ai_disclosure` and `DMRequest.ai_disclosure` both default to `True`.

Setting either to `False` raises `ValueError` in the adapter before any network call.  This is **non-negotiable** per operating rules and cannot be overridden at runtime.

## Providers — when to use which

| Provider | Use case | Status |
|----------|----------|--------|
| `mock` | Tests, CI, offline demo | ✅ Ready |
| `browser_use` | Open-ended browsing tasks | STUB (wire LLM creds) |
| `code_agent` | Bulk structured extraction | STUB (wire smolagents) |
| `skill_api` | Named browser macros | STUB (set `BROWSER_SKILL_API_BASE_URL`) |
| `platform_api` | Official REST API calls | STUB (set platform token env vars) |

## Testing

```bash
cd packages/browser_runtime
uv pip install -e ".[dev]"
uv run pytest tests/ -v --tb=short
```

Tests are fully isolated: each test gets its own settings singleton, audit logger, and tmp directory.  No credentials or network access required.
