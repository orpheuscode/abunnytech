## Context

Stage 3 defines local contract stubs in `agents/stage3_distribution/contracts.py`
because `packages/contracts` does not yet exist. When the contracts package is
created, these models should become the canonical definitions.

---

## Proposed canonical models

### `IdentityMatrix`

```python
class CommentStyle(BaseModel):
    tone: str
    use_emojis: bool
    avg_reply_length: int
    trigger_keywords: list[str]
    dm_offer_template: str
    positive_reply_templates: list[str]
    question_reply_templates: list[str]
    faq: dict[str, str]

class IdentityMatrix(BaseModel):
    identity_id: str
    persona_name: str
    display_name: str
    niche: str
    bio: str
    visual_style: str
    voice_tags: list[str]
    hashtags: list[str]
    target_platforms: list[Platform]
    comment_style: CommentStyle
    ai_disclosure_footer: str   # REQUIRED — format string with {persona_name}
    created_at: datetime
    metadata: dict[str, Any]
```

Key requirements for all stage owners:
- `ai_disclosure_footer` must remain non-nullable with a valid format string
- `comment_style.trigger_keywords` drives the DM FSM in Stage 3
- `comment_style.dm_offer_template` is the DM/reply offer text

## No breaking changes to existing contracts

Stage 3 does not modify `packages/browser_runtime` types. The `Platform` enum
in `browser_runtime.types` and the `Platform` enum in `contracts.py` are kept
in sync by value (both use `"instagram"`, `"tiktok"`, `"youtube"` strings).
The executor maps between them by `.value`.
