# Handoff: m1t2 — Stage 0 identity foundation

## Purpose

Compile authoring-time persona inputs into pipeline artifacts consumed by later stages (1–5) without calling live external APIs unless explicitly configured.

## Inputs

| Input | Description |
| --- | --- |
| Persona file | YAML (`.yaml`/`.yml`) or JSON (`.json`) validated as `PersonaSetup` in `abunny_stage0_identity.models_input` |
| CLI flags | `--input`, `--out`, `--dry-run`, optional `--matrix-id` |

### Persona schema (high level)

- `display_name`, `niche` (required)
- `demographics` (age_range, locale, gender_presentation, location_hint)
- `personality` (traits, energy, voice_description)
- `product_categories`
- `posting_cadence` (posts_per_week, best_windows_utc)
- `comment_style` (length, emoji_use, signature_phrases)
- `dm_trigger_rules` (match, action, notes)
- `visual_style` (palette, lighting, camera, wardrobe_notes, background_notes)
- `platform_targets` — must be values from `pipeline_contracts.models.enums.PlatformTarget` (e.g. `tiktok`, `shorts`, `instagram_reels`, `youtube_shorts`)
- `avoid_topics`, `disclosure_line`
- `integrations` — optional hints: `higgsfield_character_id`, `nano_banana_collection_id`, `elevenlabs_voice_id`

## Outputs (written to `--out`)

| File | Description |
| --- | --- |
| `identity_matrix.json` | `pipeline_contracts.models.identity.IdentityMatrix` (JSON) |
| `training_materials_manifest.json` | `TrainingMaterialsManifest` with `TrainingMaterialKind` values |
| `asset_manifest.json` | Local manifest (not a contracts model): avatar, voice, and Nano Banana stub rows |
| `system_prompt.md` | Markdown brief + embedded JSON snapshot of the matrix for LLM injection |

## Environment variables (future live mode)

Dry-run does **not** require keys. For future non–dry-run provisioning (stubs today), reserve:

| Variable | Used by (planned) |
| --- | --- |
| `HIGGSFIELD_API_KEY` | Higgsfield character import/generation |
| `ELEVENLABS_API_KEY` | Voice provisioning |
| `NANO_BANANA_API_KEY` (or product-specific name) | Visual asset registration |

Until those integrations are implemented, **`--dry-run` is required** unless integration hints supply external ids (avatar/voice); Nano Banana rows are always manifest stubs in dry-run.

## Contracts

- Consumes **`packages/contracts`** models only; does not modify them.
- Rich persona fields are normalized into `PersonaAxis` / `IdentityMatrix` where possible; full operational detail lives in `system_prompt.md`.

## Commands

