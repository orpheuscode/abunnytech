## What the tests cover

### `tests/test_contracts.py` (36 tests)

Validates each canonical contract can be:
- Instantiated with minimum required fields
- Round-tripped through JSON
- Rejected on invalid field values

Covers: `IdentityMatrix`, `VideoBlueprint`, `ContentPackage` (S3 + S4 variants),
`DistributionRecord` (S3 + S4), `DMConversationRecord`, `PerformanceMetricRecord`,
`OptimizationDirectiveEnvelope`, `RedoQueueItem`, `ProductCatalogItem`.

### `tests/test_handoffs.py` (31 tests)

| Class | Boundary | What it checks |
|---|---|---|
| `TestS0ToS1Handoff` | Stage 0 → 1 | IdentityMatrix has niche, platforms, voice_tags for discovery |
| `TestS1ToS2Handoff` | Stage 1 → 2 | VideoBlueprint has hook_style, topic, duration |
| `TestS2ToS3Handoff` | Stage 2 → 3 | ContentPackage accepted by scheduler + executor produces DistributionRecord |
| `TestS3ToS4Handoff` | Stage 3 → 4 | S3 record adapts to S4 schema; Stage4Runner accepts adapted records and produces directives |
| `TestFeatureFlags` | All stages | Stage 5 off by default; no live credentials; ShopifyAdapter absent from stage3 executor |
| `TestDryRunPropagation` | All stages | dry_run=True propagates through Stage 3 and Stage 4 outputs |

## S3 → S4 field mapping (known boundary seam)

Stage 3 and Stage 4 currently have divergent `DistributionRecord` schemas.
The adapter in `packages/evals/fixtures.py` handles the translation:

| Stage 3 field | Stage 4 field | Notes |
|---|---|---|
| `package_id` | `content_package_id` | Renamed |
| `platform` (enum) | `platform` (str) | `.value` coercion |
| `status` (DistributionStatus) | `status` (Literal) | Mapped via `_STATUS_MAP` |
| `record_id` | `record_id` | Same |
| `post_id` | `post_id` | Same |

When `packages/contracts` is created, both stages should import a single canonical
`DistributionRecord` that eliminates this adapter.

  `make_content_package()` in `scripts/smoke/run_smoke.py` with calls to
  your actual runner functions when those are ready. The fixture factories

- **`packages/contracts` unification**: When canonical contracts land, update
  imports in `packages/evals/fixtures.py` and remove `adapt_s3_to_s4_distribution_record()`.

  ```bash
  STAGE5_MONETIZE_ENABLED=true python scripts/smoke/run_smoke.py
  ```
  The smoke script currently skips all ShopifyAdapter calls even when the flag is on
  (mock mode only). Wire in real Stage 5 runner when that terminal is complete.
