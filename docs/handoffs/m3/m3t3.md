# Stage 5 — Monetize (Handoff)

> **Classification:** Optional / Stretch
> **Feature-flagged:** Yes — `FEATURE_STAGE5_MONETIZE=false` by default
> **Owner:** m3t3

---

## Overview

Stage 5 adds a non-blocking monetization layer to the abunnytech pipeline.
It produces product scoring, store listing drafts, brand outreach drafts, and
monetization attribution records — all in **dry-run mode** by default.

**This stage is entirely optional and must never block the core Stage 0–4 demo.**

## Feature Flag

| Env var | Default | Effect |
|---------|---------|--------|
| `FEATURE_STAGE5_MONETIZE` | `false` | All HTTP routes return 403; service methods return `{"ok": false}` |
| `DRY_RUN` | `true` | All adapters produce simulated output; no live API calls |

When `FEATURE_STAGE5_MONETIZE=false` (the default), Stage 5 is completely
inert — it registers no side effects and adds zero latency to Stages 0–4.

## Key Design Decisions

### Draft/simulated by default
Every adapter has a `Mock*` implementation that produces realistic but fake
output. Listing drafts are written to `output/stage5_drafts/` as JSON files.
No Shopify, social API, or external service is contacted unless explicitly
wired with a live adapter.

### Manual-approval-first
Any action classified as `publish_listing`, `send_outreach`, `send_dm`, or
`purchase` requires an explicit `ApprovalRequest` to be created and reviewed
before execution. The `ApprovalStatus` starts as `pending` and transitions
only via the `review_approval` endpoint.

### Attribution consumes Stage 3/4 data read-only
The attribution model reads `DistributionRecord` from Stage 3 and
`PerformanceMetricRecord` from Stage 4 via `list_pipeline_records`. It does
not write to those stages or modify their data.

## Adapter Protocols

| Protocol | Mock impl | Purpose |
|----------|-----------|---------|
| `ProductScorerAdapter` | `MockProductScorer` | Score products for relevance/audience/margin |
| `CatalogIngestAdapter` | `MockCatalogIngest` | Import products from external feeds |
| `ShopifyAdapter` | `MockShopifyAdapter` | Create/publish store listing drafts |
| `OutreachDraftGenerator` | `MockOutreachDraftGenerator` | Template brand outreach messages |
| `AttributionAdapter` | `MockAttributionAdapter` | Link monetization to upstream analytics |

To swap in a real adapter, construct `MonetizeService(scorer=RealScorer(), ...)`.

## API Endpoints

All routes live under `/monetize` and require `FEATURE_STAGE5_MONETIZE=true`.

| Method | Path | Description |
|--------|------|-------------|
| POST | `/monetize/products` | Add a product to the catalog |
| GET | `/monetize/products` | List products |
| POST | `/monetize/products/ingest` | Ingest products from external source |
| POST | `/monetize/products/score` | Score a single product |
| POST | `/monetize/products/score-all` | Score all products for an identity |
| POST | `/monetize/listings/draft` | Create a Shopify listing draft |
| GET | `/monetize/listings/drafts` | List listing drafts |
| POST | `/monetize/outreach` | Create a brand outreach record |
| GET | `/monetize/outreach` | List outreach records |
| POST | `/monetize/outreach/draft` | Generate an outreach draft message |
| GET | `/monetize/outreach/drafts` | List outreach drafts |
| POST | `/monetize/dm` | Log a DM conversation |
| POST | `/monetize/attribution` | Compute monetization attribution |
| POST | `/monetize/approvals` | Request manual approval |
| POST | `/monetize/approvals/review` | Review an approval request |
| GET | `/monetize/approvals` | List approval requests |

## Known Limitations

- No real Shopify API integration yet — only mock adapter.
- Outreach drafts use template strings, not LLM-generated content.
- Attribution model is heuristic-based; a production version would need
  proper click-tracking and conversion funnels.
- Approval workflow is synchronous (poll-based); a production version
  should support webhooks or push notifications.

## Future Work
