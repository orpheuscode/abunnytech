# m3t2 Handoff -- Pipeline Dashboard

## Startup Steps

1. **Install dependencies** (from project root):
   ```bash
   pip install -e ".[dev]"
   ```
   Or just the dashboard deps:
   ```bash
   pip install streamlit httpx pandas
   ```

2. **Run the dashboard**:
   ```bash
   streamlit run apps/dashboard/app.py
   ```
   Opens at `http://localhost:8501`.

3. **Data source**: By default the dashboard loads fixture JSON (`apps/dashboard/fixtures/demo_seed.json`). To connect to the live API, switch the sidebar radio to "Local API" and enter the control plane URL (default `http://localhost:8000`).

## Pages

| Page | Stage | What it shows |
|------|-------|---------------|
| Identity | 0 | Persona card with voice, avatar, guidelines, platform presence |
| Discovery | 1 | Trending audio, competitor watch list, training manifest |
| Content | 2 | Video blueprints with scene breakdown, content packages with assets |
| Distribution | 3 | Distribution records table, status metrics, timeline details |
| Analytics | 4 | KPI metrics, optimization directive envelopes, redo queue |
| Monetization | 5 | Product catalog, brand outreach records (feature-flagged) |

## Demo Features

### Dry-Run Demo
From the **Demo Control** page, click "Run Dry-Run Demo" to simulate a full pipeline pass with animated progress through all 8 stages.

### Artifact Inspector
Select any contract type and drill into the raw JSON of any record. Useful for judges who want to see the data structures.

### Guided Demo
The **Guided Demo** page walks through the pipeline story in 8 steps with Previous/Next navigation. Each step renders the real page content with narrative context. Recommended flow for a recorded demo:

1. Overview -- pipeline architecture
2. Identity -- who the AI creator is
3. Discovery -- trends and competitors
4. Content -- blueprints and rendered packages
5. Distribution -- dry-run and live posting
6. Analytics -- metrics, directives, and feedback loop
7. Monetization -- products and brand deals
8. Finale -- the autonomous loop explained

## Architecture Notes

- **No business logic**: The dashboard only consumes API/contracts via `data_loader.py`. It never imports stage service code directly.
- **Fixture fallback**: `data_loader.py` tries the API first, then falls back to `demo_seed.json`. This means the dashboard works without any backend running.
- **No credentials needed**: The dashboard has no platform API keys or secrets. It reads data only.
- **Feature flags**: The Monetization page renders regardless of the `FEATURE_STAGE5_MONETIZE` flag since it only displays data. The flag controls whether stage 5 *produces* data.
