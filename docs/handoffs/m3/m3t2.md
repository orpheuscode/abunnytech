# m3t2 Handoff -- Pipeline Dashboard

## Startup Steps

1. **Install dependencies** (from project root):
   ```bash
   uv sync
   ```

2. **Run the dashboard**:
   ```bash
   uv run python -m runtime_dashboard.flask_owner_app
   ```
   Or start the full demo (API + control plane + dashboard):
   ```bash
   uv run python scripts/demo.py
   ```
   Opens at `http://localhost:8501` (see `DASHBOARD_PORT`).

3. **Data source**: By default the dashboard loads fixture JSON (`runtime_dashboard/fixtures/demo_seed.json`). To connect to the live API, use the sidebar **Data source** form: choose **Local API** and set the base URL (default `http://localhost:8000`).

## Pages

| Page | Stage | What it shows |
|------|-------|---------------|
| Identity | 0 | Persona card with voice, avatar, guidelines, platform presence |
| Discovery | 1 | Trending audio, competitor watch list |
| Content | 2 | Video blueprints with script, content packages |
| Distribution | 3 | Distribution records, status, dry-run badges |
| Analytics | 4 | Optimization directive envelopes, redo queue |
| Monetization | 5 | Product catalog (feature-flagged backend) |

## Demo Features

### Dry-Run Demo
From the **Demo Control** page, use **Run dry-run demo** for a simulated full pipeline pass (flash confirmation).

### Artifact Inspector
Select a contract type and record index to view raw JSON.

### Guided Demo
The **Guided Demo** page walks through the pipeline story in steps with links to full stage views.

### API keys
Use **API keys** in the sidebar to persist OpenAI (Browser Use), Gemini/Google, and Twelve Labs keys to `runtime_dashboard/.owner_secrets.json` (merged into `scripts/demo.py` child processes).

## Architecture Notes

- **No business logic**: The dashboard consumes data via `data_loader.py`. It does not import stage service code directly.
- **Fixture fallback**: `data_loader.py` uses the API when configured, otherwise fixture JSON. The UI works without a backend when using fixtures.
- **Feature flags**: The Monetization page displays catalog data from fixtures or API; `FEATURE_STAGE5_MONETIZE` controls whether stage 5 produces live data.
